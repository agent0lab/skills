# Python: request and pay (x402)

SDK 1.7.0+. Use `sdk.request(options)` or `sdk.fetchWithX402(options)` for HTTP requests with built-in 402 handling. On 2xx the SDK returns the parsed body (default JSON); on 402 it returns a structured object instead of raising.

## Basic flow

```python
result = sdk.request({"url": "https://api.example/paid", "method": "GET"})

if getattr(result, "x402Required", False) or (isinstance(result, dict) and result.get("x402Required")):
    # Server wants payment. result.x402Payment.accepts lists options.
    paid = result.x402Payment.pay(0)  # pay with first accept
    # paid is the same shape as a normal 2xx response
else:
    # Success: result is the parsed response body
```

Use `isX402Required(result)` from `agent0_sdk` for a type guard.

## Options dict

- **url**, **method** — Required.
- **headers** — Optional.
- **body** — Optional.
- **payment** — Optional. Payment payload to send with the first request (see x402-payment-on-first-request.md).
- **parse_response** / **parseResponse** — Optional. Callable to parse 2xx body; default is JSON.

## pay() and pay_first()

- **x402Payment.pay(accept=None)** — `accept` can be index (int), single accept, or omitted (single accept). Builds EVM payment, retries request with PAYMENT-SIGNATURE, returns parsed success value.
- **x402Payment.pay_first()** — When SDK provides balance checking, pays using first accept with sufficient balance. May be `None` if not available.

Requires SDK configured with signer for payment. Non-EVM accepts are filtered out when building payment.

## Example (from demo)

```python
result = sdk.request({"url": x402_search_url, "method": "GET"})
if isX402Required(result):
    paid = result.x402Payment.pay(0)
    print(json.dumps(paid, indent=2))
else:
    print(json.dumps(result, indent=2))
```
