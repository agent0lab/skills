# TypeScript: request and pay (x402)

SDK 1.7.0+. Use `sdk.request()` or `sdk.fetchWithX402()` for HTTP requests with built-in 402 handling. On 2xx the SDK returns the parsed body (default JSON); on 402 it returns a structured object instead of throwing.

## Basic flow

```ts
const result = await sdk.request({ url: 'https://api.example/paid', method: 'GET' });

if (result.x402Required) {
  // Server wants payment. result.x402Payment.accepts lists options.
  const paid = await result.x402Payment.pay(0);  // pay with first accept
  // paid is the same shape as a normal 2xx response
} else {
  // Success: result is the parsed response body
}
```

## Types

- **X402RequestResult&lt;T&gt;** — Either `T` (success) or **X402RequiredResponse&lt;T&gt;** (402).
- **X402RequiredResponse&lt;T&gt;** — `{ x402Required: true, x402Payment: X402Payment&lt;T&gt; }`.
- **X402Payment&lt;T&gt;** — `accepts: X402Accept[]`, plus `pay(accept?: X402Accept | number): Promise<T>`, and optional `payFirst(): Promise<T>`.
- **X402Accept** — One payment option: `price`, `token`, `network`, `destination`, etc.

Type guard: `isX402Required(result)` narrows to 402 response.

## pay() and payFirst()

- **pay(accept?)** — No arg: use single accept; number: `accepts[index]`; or pass an `X402Accept`. Builds payment (EIP-3009-style for EVM), retries request with payment header, returns parsed success value.
- **payFirst()** — When SDK was built with balance checking, pays using the first accept for which the signer has sufficient balance. Throws if none.

Requires SDK configured with signer (private key or wallet) for payment. Non-EVM accepts are filtered out when building payment.

## Example (from demo)

```ts
const result = await sdk.request({ url: X402_SEARCH_URL, method: 'GET' });
if (result.x402Required) {
  const paid = await result.x402Payment.pay(0);
  console.log(JSON.stringify(paid, null, 2));
} else {
  console.log(JSON.stringify(result, null, 2));
}
```
